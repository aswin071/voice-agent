MVP Model for a Production Voice Agent

The minimum viable voice agent that can be deployed to real customers has 5 non-negotiable layers. Every layer below must work reliably before moving to Phase 2.

🎙
1 · Audio I/O
Clean microphone capture, VAD to detect speech vs silence, noise cancellation, and low-latency audio output back to caller.
VAD Noise Cancel <200ms
📝
2 · STT
Streaming speech-to-text with correct language/dialect. Word Error Rate <15% on target domain vocabulary. Turn endpointing.
Sarvam saaras:v3 streaming
🧠
3 · LLM Reasoning
Fast LLM with well-scoped system prompt, conversation history, and function tools for handoffs. Latency <1s first token.
Claude Haiku tools
🔊
4 · TTS
Natural-sounding voice in the caller's language. Streaming synthesis so audio starts playing before full text is ready.
Sarvam bulbul:v3 streaming
🔁
5 · Session Mgmt
Conversation state across turns, graceful error recovery, call timeout, barge-in support (interrupt agent mid-speech).
LiveKit AgentSession

What the MVP agent must handle reliably

✓ Must-Have Behaviours
01
Happy path works end-to-end
Caller speaks → agent understands → collects data → confirms → closes call.
02
Handles "I didn't understand"
Agent re-asks politely (max 2 retries) before graceful fallback.
03
Barge-in (caller can interrupt)
Agent stops speaking immediately when caller starts talking.
04
Silence timeout
After 8–10s silence, agent prompts; after 20s, call ends gracefully.
05
Out-of-scope deflection
Agent politely declines topics outside its task and re-steers.
06
Single confirmed output
At minimum: a reference number or confirmation SMS triggers downstream.
✗ NOT required in MVP
—
Real-time CRM write-back
Log to a Google Sheet or simple DB first. CRM API comes later.
—
Outbound calling
Inbound only is a complete product. Outbound needs SIP + dialer logic.
—
Live dashboard with real data
Mock data UI is fine. Ship the working agent first.
—
Supervisor barge / whisper
Advanced telephony feature. Not needed to validate the concept.
—
Analytics & reporting
Call logs with timestamp + outcome are enough. Dashboards come after.
—
Fine-tuned / custom LLM
Claude/GPT with a good system prompt beats a bad fine-tune every time.
